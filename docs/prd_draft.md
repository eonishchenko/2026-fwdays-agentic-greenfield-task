Цей проєкт оптимізує час на заповнення документів використовуючи шаблони та завантаження його у pdf та json документы.
Мае наступну функціональність
- переглянути створений документ, перехід на останню не заповненний крок флоу або у кінец - `url/docs/guid`
    - Має дві кнопки, завантажити як `pdf`
    - Редагувати (перехід на флоу створення)

Этап заповнення документа
- флоу з покроковим заповненням даних та відображенням поточної сторінки
    - створюєтся `guid` під капотом це файл `./docs/guid.json`
    - кожен крок має дві кнопки для переходу `назад` або `далі`
    - флоу має постійний http урл `/docs/guid`
    - first step:
        - користувач обирає тип документа
            - рахунок + акт (default)
            - рахунок
            - акт
        - або вводить `guid` документа для копіювання всіх данних окрім номера документа.
    - second step:
        - дата документа проставляется для двох документів з можливістю редагування
        - поточний день — date by default
        - Номер документа для рахунока проставляется як `Р-{0000001}` з довжиною у 7 цифр
        - Номер документа для акта проставляется як `{1}`
        - Остання згенерована цифра зберігается `./{year}/doc_number.json` де `year` рік документа
    - third step:
        - Вибрати данні замовника або створити контакт
        - Контакт має зберігатися по РНОКПП `./contacts/{РНОКПП}.json`
        - поля:
            - ФІО -> `full-name`
            - РНОКПП -> `inn`
            - телефон -> `phone`
            - рахунок -> `acc`
            - банк -> `bank`
            - МФО -> `mfo-bank`
            - адреса -> `addr`
    - fourth step:
        - Вибрати дані виконавця або створити контакт
        - Контакт має зберігатися по РНОКПП `./contacts/{РНОКПП}.json`
        - поля:
            - ФІО -> `full-name`
            - РНОКПП -> `inn`
            - телефон -> `phone`
            - рахунок -> `acc`
            - банк -> `bank`
            - МФО -> `mfo-bank`
            - адреса -> `addr`
    - fifth step:
        - Вибрати каталог послуг або створити новий
        - Список послуг має зберігатися `./jobs/services.json`
        - поля:
            - коротка назва -> `sign-name`
            - Назва для документа -> `service-name`
            - Кількість -> `amount`
            - Ціна за послугу -> `price`
        - Опція додати ще послуги
    - sixth step:
        - Заповнити шаблон для рахунка з `./templates/invoce.html`
            - замовника - поля з префіксом `client-`
            - виконавця - поля з префіксом `done-by-`
            - сумму прописом - поля `amount-cursive` Приклад: `8750` буде записано як `Вісім тисяч сімсот п'ятдесят гривень 00 копійок`
            - дата - `date`. Приклад `15.05.2021`
            - дата прописом - поля `date-cursive`. Приклад `15.05.2021` буде записано як `15 травня 2021 р.`
            - послуга - поле `service-cursive`
            - кількість - поле `done_amount`
            - ціна - поля `price`
            - підсумок сумм усіх полів - поле `price-total`
        - Заповнити шаблон для акта з `./templates/complete.html`
            - замовника - поля з префіксом `client-`
            - виконавця - поля з префіксом `done-by-`
            - сумму прописом - поля `amount-cursive`. Приклад: `8750` буде записано як `Вісім тисяч сімсот п'ятдесят гривень 00 копійок`
    - seven step:
      - сформовані документи без можливості редагування
      - напроти кожного готового документа є кнопка завантажити у pdf
      - є кнопка повернутися на попередній крок
- Дізайн
  - Нейтральні кольори для фінансових документів
  - Відображати прогрес кроків у вигляді цифр у кружках
- Helpers
    - конвертація у суми прописом. Приклад: `8750` буде записано як `Вісім тисяч сімсот п'ятдесят гривень 00 копійок`
    - конвертація дати прописом. Приклад `15.05.2021` буде записано як `15 травня 2021 р.`
